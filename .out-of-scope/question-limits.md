# Grilling 期间的问题数量硬限制

`/grill-me` skill（以及其他 skills 内部的 grilling session）不会强制设置最大问题数。增加可配置上限或硬性封顶不在范围内。

## 为什么这不在范围内

Grilling 有意保持开放式。它的目标是持续深挖，直到 decision tree 的每个分支都被解决：有些计划只需要三个问题，有些需要五十个。固定上限要么会在困难问题上过早截断有价值的探索，要么会在简单问题上显得武断。

如果一次 session 感觉太长，已经有合适的逃生口：

- 用户可以随时停止 session，并接受当前计划状态。
- 用户可以告诉模型收尾、总结并继续；自然语言引导是预期的控制面，而不是数字限制。

增加硬上限也会混淆两种不同失败模式：模型因为计划确实欠明确而问很多问题（按预期工作），以及模型问了重复或低价值的问题（prompt 质量问题，不是数量问题）。后者的修复应该放在 skill prompt 中，而不是计数器里。
